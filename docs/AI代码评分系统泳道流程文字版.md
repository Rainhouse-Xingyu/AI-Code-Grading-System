# AI 代码评分系统泳道流程文字版

## 泳道划分

建议按业务责任主体划分泳道：

- 系统管理员
- 教师
- 学生
- AI 评分服务

说明：后端系统、数据库、文件存储属于技术实现承载，不单独作为角色泳道；相关处理写在对应步骤中。

## 主流程

1. 系统管理员维护全局评分模板、评分点和模型运行参数。
2. 教师导入学生名单，系统创建学生账号并生成初始密码。
3. 教师创建作业，填写标题、描述、编程语言、截止时间、迟交策略，并选择发布班级和评分点。
4. 教师判断是否发布作业。
5. 如果暂不发布，作业保持草稿状态，教师继续修改。
6. 如果确认发布，学生端可以看到该作业。
7. 学生登录系统，查看已发布作业，选择作业后上传 ZIP 代码包。
8. 系统校验提交文件和提交时间，包括文件类型、截止时间和迟交策略。
9. 如果校验不通过，学生收到提示并重新提交，或因截止策略被禁止提交。
10. 如果校验通过，系统保存 ZIP 文件，并按“学号_姓名.zip”规则重命名。
11. 系统创建提交记录，解压 ZIP，递归读取代码文件，生成结构化代码 JSON。
12. 系统将提交状态更新为已解析，学生看到“已提交，等待教师评分”。
13. 教师查看作业提交列表，筛选需要评分的提交记录。
14. 教师勾选提交记录并批量发起 AI 评分。
15. 系统创建 AI 评分任务，将提交状态更新为评分中。
16. AI 评分服务领取任务，读取结构化代码 JSON 和评分标准 JSON。
17. AI 评分服务构建 Prompt，优先调用 DeepSeek 进行评分。
18. 如果 DeepSeek 不可用，AI 评分服务切换本地模型；如果本地模型也不可用，则使用规则兜底评分。
19. 如果评分任务失败，系统记录失败日志，教师可以重试失败任务。
20. 如果评分成功，AI 评分服务返回总分、分项得分、问题分析、改进建议和 Markdown 报告。
21. 系统保存 AI 报告和任务日志，并将提交状态更新为已评分。
22. 教师查看 AI 报告、评分明细、历史报告或模型对比结果。
23. 教师进行人工复核，可以调整分数、评语和报告内容。
24. 系统保存教师复核记录，并将提交状态更新为已复核。
25. 教师选择单个、多个或当前作业全部已复核提交，发布成绩。
26. 系统发布成绩；如果提交为迟交且作业设置了扣分策略，系统在发布时计算最终扣分。
27. 系统写入成绩发布记录，并将提交状态更新为已发布。
28. 学生查看已发布成绩、教师最终评分和最终 Markdown 报告。
29. 教师按需导出单份报告、批量 PDF、提交包、报告包或成绩表 Excel。

## 异常与回退流程

- 作业未发布：学生不可见，教师可继续编辑。
- ZIP 校验失败：学生重新上传；如果迟交策略为禁止迟交，则系统拒绝提交。
- 提交进入评分、已评分、已复核或已发布后：学生不能再次覆盖提交。
- AI 评分失败：任务状态变为失败，教师可手动重试。
- 教师复核后未发布：学生仍不能看到最终成绩。
- 成绩已发布后需要修改：教师必须先撤回成绩，再修改复核内容。
- 教师可将提交打回重交，系统会删除该提交关联的文件、AI 报告、任务日志和发布记录。

## 精简版流程

系统管理员维护模板和模型参数 → 教师导入学生并创建作业 → 教师发布作业 → 学生上传 ZIP → 系统校验、保存并解析代码 → 教师批量发起 AI 评分 → AI 评分服务生成评分报告 → 教师复核并调整结果 → 教师发布成绩 → 学生查看最终成绩和报告 → 教师导出报告或成绩表。
